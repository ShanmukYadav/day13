## 📝 Part A — `survey_cleaner.py` (40%)

End-to-end cleaning pipeline on a 55-row customer satisfaction survey dataset with **10 deliberate data quality issues**.

### Data Quality Issues Embedded

| # | Issue | Example |
|---|---|---|
| 1 | Real NaN / None values | `purchase_amount = None` |
| 2 | Empty string as missing | `gender = ""` |
| 3 | Hidden NaN strings | `"N/A"`, `"null"`, `"None"`, `"n/a"` |
| 4 | Duplicate rows | Row 52–53 repeat rows 0–1 |
| 5 | Wrong dtypes | `age`, `score` stored as strings |
| 6 | Inconsistent casing | `"Electronics"`, `"ELECTRONICS"`, `"electronics"` |
| 7 | Extra whitespace | `"  New York  "`, `" Male"` |
| 8 | Invalid numeric values | `age = -5`, `age = 200`, `score = 0`, `score = 11` |
| 9 | Mixed date formats | `"15/03/2024"`, `"2024-03-15"`, `"22-May-2024"` |
| 10 | Inconsistent binary labels | `"Yes"`, `"yes"`, `"YES"`, `"NO"` |

### Key Functions

**`detect_issues(df)`** — returns a dict with:
- `total_rows`, `total_columns`
- `total_missing`, `missing_per_column`
- `duplicate_count`
- `wrong_types` (object columns that should be numeric)
- `invalid_values` (age out of range, score out of range, negative purchase)

**`clean_data(df)`** — 6-step pipeline:
1. Replace hidden NaN markers with real `np.nan`
2. Fix dtypes with `to_numeric(errors='coerce')` and `to_datetime(dayfirst=True, errors='coerce')`
3. Standardise text: `.str.strip()` → `.str.lower()` → collapse multiple spaces
4. Fill missing values (strategy justified per column — see comments in code)
5. Drop unfixable rows (invalid age/score/amount, missing name/date)
6. Remove duplicates

### Fill Strategy Justification

| Column | Strategy | Why |
|---|---|---|
| `age` | Median | Right-skewed; median is robust to outliers |
| `satisfaction_score` | Median | Ordinal 1–10 scale; median preserves rank order |
| `purchase_amount` | Median | Money data is right-skewed; median > mean |
| `gender` | Mode | Categorical — most frequent label is best guess |
| `city` | Mode | Categorical — same reasoning |
| `product_category` | Mode | Categorical — same reasoning |
| `would_recommend` | Mode | Binary categorical — fill with majority class |
| `respondent_name` | Drop row | Missing name = unidentifiable row |
| `survey_date` | Drop row | Time-series analysis needs valid date; imputing is risky |

## 📝 Part B — `data_profiler.py` (30%)

A **reusable** `profile_dataframe(df)` function that works on any DataFrame.

### Per-column output

| All columns | Numeric extras | String extras |
|---|---|---|
| dtype | min, max, mean | avg/min/max length |
| unique count | median, std | whitespace issues |
| null count + % | skewness | mixed case detection |
| top-5 values | outliers (>3σ) | — |

### Issue detection
- **Single-value columns** — zero variance, useless for ML
- **High-cardinality strings** — >50% unique, likely free-text or ID columns
- **Numeric outliers** — values more than 3 standard deviations from the mean

### Self-study APIs used
- `df.describe(include='all')`
- `df.nunique()`
- `df.memory_usage(deep=True)`
- `scipy.stats.skew()` (falls back to `pd.Series.skew()` if scipy not installed)

## 📝 Part C — `interview_answers.py` (20%)

### Q1 — 1M rows, 40% of `income` missing

Decision framework: MCAR → MAR → MNAR → choose drop or fill strategy.
**Recommendation:** Group-wise median fill (by age_group/region) + add `income_was_missing` indicator column. Do not drop — losing 400K rows biases the dataset.

### Q2 — `standardize_column(series)`

Cleans a messy text Series in one chain:
```python
series.str.strip().str.lower()
      .str.replace(r"[^a-z0-9\s]", "", regex=True)
      .str.replace(r"\s+", " ", regex=True).str.strip()
```

Test results:
```
' Hello World!! '  →  'hello world'
' NEW YORK '       →  'new york'
'san--francisco'   →  'sanfrancisco'
' MUMBAI '         →  'mumbai'
```

### Q3 — 4 Bugs Fixed

| Bug | Original | Fix |
|---|---|---|
| **Bug 1** | `to_numeric` before stripping commas — `"1,500"` silently becomes `NaN` | Strip commas + replace hidden NaN markers first |
| **Bug 2** | Python `and` on Series → `ValueError` | Use `&` with parentheses |
| **Bug 3** | `.str.contains()` without `na=False` or `case=False` → `TypeError` + misses casing variants | Add `na=False, case=False` |
| **Bug 4** | `to_datetime()` without `dayfirst=True` or `errors='coerce'` → `ParserError` on mixed formats | Add `dayfirst=True, errors='coerce'` |

## 🧠 Key Concepts Covered

- **`isna()` / `dropna()` / `fillna()`** — detecting and handling real NaN values
- **Hidden NaN markers** — `"N/A"`, `"null"`, `""` — must be replaced before any analysis
- **`to_numeric(errors='coerce')`** — safer than `astype()` for untrusted data
- **`to_datetime(dayfirst=True, errors='coerce')`** — handles mixed date formats without crashing
- **`.str` accessor** — `.strip()`, `.lower()`, `.replace()`, `.contains()`, `.len()`
- **`drop_duplicates()`** — exact row deduplication
- **Fill strategy selection** — mean vs median vs mode vs drop, justified per column type
- **`memory_usage(deep=True)`** — accurate memory profiling for object columns
- **Data profiling** — skewness, outlier detection, cardinality analysis
